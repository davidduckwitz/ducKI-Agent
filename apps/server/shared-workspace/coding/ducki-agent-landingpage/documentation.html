DucKI Agent bedienen

Diese Seite zeigt dir den schnellsten Weg vom ersten Start bis zur produktiven Nutzung im Alltag. DucKI ist kostenlos nutzbar, self-hosted und offen fuer Pull Requests.

1. Quickstart Setup

Windows (PowerShell)

pnpm install
Copy-Item .env.example .env
pnpm dev

Linux/macOS/WSL

pnpm install
cp .env.example .env
pnpm dev
  • Web UI: http://localhost:3000
  • API: http://localhost:3001
  • Healthcheck: http://localhost:3001/health

2. Erste Konfiguration

Settings

In /settings den Provider und das Modell setzen (z. B. LM Studio, OpenRouter, OpenAI, Ollama).

Skills

In /skills nur die Skills aktivieren, die du wirklich brauchst. Das spart Kontext und macht den Agenten praeziser.

3. Bedienung im Alltag

Chat

/chat fuer iterative Arbeit, Recherche, Planung und Tool-gestuetzte Umsetzung.

Workflow

/workflow fuer graph-basierte, mehrstufige Ausfuehrungen mit Resume-Faehigkeit.

Memory

/memory fuer Profile, Eintraege, Freigaben und kontrollierte Langzeit-Kontextpflege.

Agents Live

/agents zur Beobachtung laufender Agent-Runs und Runtime-Aktivitaet.

4. Coding, Tasks und Automatisierung

  • /coding fuer projektgebundenes Arbeiten mit Dateiexplorer, Editor, Preview und Chat-Kontext.
  • /tasks fuer planbare Arbeitsitems inkl. Agent-Ausfuehrung pro Task.
  • /cronjobs fuer zeitgesteuerte Prompts, Tasks, Skills und Tool-Runs.
  • /shared fuer gemeinsame Artefakte und Uploads.

5. Troubleshooting Kurzguide

Provider antwortet nicht

Pruefe in /settings Provider-URL, Modellname und API-Key. Danach Chat erneut starten.

Feature fehlt im Menue

Flags wie CODING_ENABLED oder WIKI_ENABLED in /settings aktivieren und UI neu laden.

Dateioperationen schlagen fehl

Nur relative Pfade im Projekt nutzen und auf den freigegebenen Workspace-Bereich achten.

Monitoring

/logs und /agents helfen bei Analyse von Fehlern und laufenden Prozessen.
